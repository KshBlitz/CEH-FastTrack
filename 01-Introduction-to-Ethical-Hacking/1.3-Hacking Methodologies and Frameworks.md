# 2. Core Concepts

## Hacking Methodology

Concept → Structured approach used to perform security assessment or cyberattack simulation   
Purpose → Ensure repeatable, systematic, and comprehensive testing    
Key Detail → Follows phased attack progression from reconnaissance to reporting

## Ethical Hacking Lifecycle

Concept → Ordered sequence used during authorized security testing    
Purpose → Simulate attacker behavior while preserving legal boundaries    
Key Detail → Reconnaissance → Scanning → Gaining Access → Maintaining Access → Clearing Tracks → Reporting

## Reconnaissance

Concept → Information gathering phase   
Purpose → Identify attack surface and potential targets   
Key Detail → Passive or active techniques used before exploitation

## Scanning

Concept → Technical discovery of hosts, services, vulnerabilities, and ports    
Purpose → Identify accessible attack vectors    
Key Detail → Includes port scanning, service detection, OS fingerprinting

## Enumeration

Concept → Extraction of detailed information from discovered services   
Purpose → Obtain usernames, shares, resources, network details    
Key Detail → More intrusive than basic scanning

## Exploitation

Concept → Abuse of vulnerabilities to gain unauthorized access    
Purpose → Validate exploitability and impact    
Key Detail → May involve privilege escalation or remote code execution

## Maintaining Access

Concept → Establishing persistence within compromised systems   
Purpose → Ensure continued control over target environment    
Key Detail → Backdoors, rootkits, scheduled tasks, persistence agents

## Clearing Tracks

Concept → Concealing attacker activity    
Purpose → Avoid detection and forensic analysis   
Key Detail → Includes log manipulation and anti-forensics techniques

## Cyber Kill Chain

Concept → Lockheed Martin attack lifecycle model    
Purpose → Understand adversary progression    
Key Detail → Seven-stage attack sequence used for detection mapping

## MITRE ATT&CK

Concept → Knowledge base of adversary tactics and techniques    
Purpose → Standardize attacker behavior analysis    
Key Detail → Maps real-world TTPs across attack stages

## MITRE D3FEND

Concept → Defensive countermeasure knowledge graph    
Purpose → Map security controls against ATT&CK techniques   
Key Detail → Defensive complement to ATT&CK

## Diamond Model of Intrusion Analysis

Concept → Threat intelligence analysis model    
Purpose → Correlate adversary infrastructure and victim relationships   
Key Detail → Adversary, capability, infrastructure, victim

## OSSTMM

Concept → Open Source Security Testing Methodology Manual   
Purpose → Standardized operational security testing   
Key Detail → Emphasizes measurable and repeatable testing

## PTES

Concept → Penetration Testing Execution Standard    
Purpose → Define penetration testing process standards    
Key Detail → Covers pre-engagement to reporting phases

## OWASP Testing Methodology

Concept → Web application testing framework   
Purpose → Standardized web security assessment    
Key Detail → Closely aligned with OWASP Top 10 risks

## AI-Driven Ethical Hacking

Concept → Use of AI/ML in offensive and defensive workflows   
Purpose → Improve automation, analysis, anomaly detection, and predictive modeling    
Key Detail → Used in vulnerability discovery, behavioral analytics, phishing simulation, and automated exploitation support

# 3. Definitions & Classifications

## Reconnaissance Types

### Passive Reconnaissance

Category → Non-intrusive information gathering    
Characteristics → No direct interaction with target   
Examples → WHOIS, search engines, social media, DNS records, public documents

### Active Reconnaissance

Category → Direct target interaction    
Characteristics → Detectable by defensive systems   
Examples → Ping sweeps, port scans, traceroute, banner grabbing

---

## Scanning Types

### Port Scanning

Purpose → Identify open ports/services    
Characteristics → TCP/UDP analysis    
Example → SYN scan using Nmap

### Vulnerability Scanning

Purpose → Detect weaknesses and CVEs    
Characteristics → Automated assessment    
Example → Nessus/OpenVAS scan

### Network Scanning

Purpose → Discover hosts and topology   
Characteristics → IP range discovery    
Example → ICMP sweep

### Banner Grabbing

Purpose → Service identification    
Characteristics → Retrieves service/version information   
Example → HTTP/FTP banner analysis

---

## Enumeration Categories

### NetBIOS Enumeration

Purpose → Extract shared resources and usernames    
Example → Enumerating SMB shares

### SNMP Enumeration

Purpose → Gather device/network information   
Example → Community string extraction

### LDAP Enumeration

Purpose → Extract Active Directory information    
Example → User/group discovery

### DNS Enumeration

Purpose → Discover subdomains and records   
Example → Zone transfer attempts

---

## Access Attack Types

### Password Attacks

Characteristics → Credential compromise attempts    
Examples → Brute force, dictionary, rainbow tables, credential stuffing

### Privilege Escalation

Characteristics → Gain higher privileges after compromise   
Examples → Kernel exploits, token impersonation

### Remote Exploitation

Characteristics → Exploit network-facing service    
Examples → RCE vulnerabilities

### Social Engineering

Characteristics → Human manipulation attacks    
Examples → Phishing, vishing, baiting

---

## Persistence Mechanisms

### Backdoor

Purpose → Hidden remote access    
Example → Reverse shell listener

### Rootkit

Purpose → Conceal malicious processes/files   
Example → Kernel-level stealth malware

### Scheduled Task Persistence

Purpose → Re-execution after reboot/login   
Example → Cron jobs / Task Scheduler abuse

### Service Persistence

Purpose → Maintain startup execution    
Example → Malicious Windows service

---

## Anti-Forensics Techniques

### Log Clearing

Purpose → Remove evidence   
Example → Event log deletion

### Timestamp Manipulation

Purpose → Alter forensic timelines    
Example → Timestomping

### File Concealment

Purpose → Hide malicious artifacts    
Example → NTFS Alternate Data Streams

### Encryption/Obfuscation

Purpose → Evade analysis    
Example → Packed malware

# 4. Differentiation Blocks

## Reconnaissance vs Scanning vs Enumeration

Reconnaissance

* Initial information gathering
* Broad attack surface mapping
* Passive or active

Scanning

* Technical probing of systems
* Detects ports/services/OS
* More direct interaction

Enumeration

* Extracts detailed system/service information
* Higher interaction level
* Often authenticated or protocol-specific

---

## Vulnerability Assessment vs Penetration Testing

Vulnerability Assessment

* Finds weaknesses
* Usually automated
* Non-exploit focused

Penetration Testing

* Exploits weaknesses
* Manual validation included
* Demonstrates impact

---

## MITRE ATT&CK vs Cyber Kill Chain

Cyber Kill Chain

* Linear attack progression
* Seven stages
* Focus on attack lifecycle

MITRE ATT&CK

* Matrix of attacker tactics and techniques
* Real-world adversary behaviors
* Much deeper operational detail

---

## MITRE ATT&CK vs MITRE D3FEND

MITRE ATT&CK

* Offensive behavior knowledge base
* Attacker-focused
* Maps TTPs

MITRE D3FEND

* Defensive countermeasure framework
* Defender-focused
* Maps mitigations and detection logic

---

## OSSTMM vs PTES vs OWASP

OSSTMM

* Operational security testing methodology
* Metrics-driven
* Broad infrastructure focus

PTES

* End-to-end penetration testing standard
* Covers engagement lifecycle
* Enterprise pentest workflow focus

OWASP Testing Guide

* Web application-focused
* Application security methodology
* Maps web attack categories

---

## Red Team vs Purple Team

Red Team

* Offensive simulation
* Adversary emulation
* Stealth-focused

Purple Team

* Collaborative improvement process
* Red + Blue integration
* Detection optimization focus

## MITRE ATT&CK Tactical Flow

Initial Access
→ Execution
→ Persistence
→ Privilege Escalation
→ Defense Evasion
→ Credential Access
→ Discovery
→ Lateral Movement
→ Collection
→ Exfiltration
→ Impact

Recognition Clues:

* Initial Access → phishing, exposed services, drive-by compromise
* Execution → PowerShell, scripts, malicious binaries
* Persistence → registry keys, startup folders, scheduled tasks
* Privilege Escalation → token abuse, kernel exploits
* Defense Evasion → obfuscation, disabling security tools
* Credential Access → keylogging, LSASS dumping, credential dumping
* Discovery → network mapping, account enumeration
* Lateral Movement → SMB, RDP, Pass-the-Hash
* Collection → staged sensitive data gathering
* Exfiltration → outbound transfer to attacker infrastructure
* Impact → ransomware, destruction, service disruption

# 9. MCQ Traps & Confusions

## Trap

“Scanning and enumeration are identical.”

Why Confusing
Both involve target interaction.

Correct Distinction
Scanning discovers systems/services; enumeration extracts detailed information from them.

---

## Trap

“Cyber Kill Chain and MITRE ATT&CK are interchangeable.”

Why Confusing   
Both model attack behavior.

Correct Distinction   
Kill Chain models attack progression; ATT&CK catalogs tactics and techniques.

---

## Trap

“Reconnaissance always touches the target system.”

Why Confusing   
Recon commonly associated with scans.

Correct Distinction   
Passive reconnaissance avoids direct interaction.

---

## Trap

“Clearing tracks occurs before exploitation.”

Why Confusing   
Students confuse anti-forensics timing.

Correct Distinction   
Clearing tracks occurs after gaining access and performing actions.

---

## Trap

“Persistence means privilege escalation.”

Why Confusing   
Both occur after compromise.

Correct Distinction   
Persistence maintains access; privilege escalation increases permissions.

---

## Trap

“OWASP is a general network penetration testing framework.”

Why Confusing   
OWASP appears in pentesting discussions.

Correct Distinction   
OWASP focuses primarily on web application security.

---

## Trap

“MITRE D3FEND replaces ATT&CK.”

Why Confusing   
Both are MITRE frameworks.

Correct Distinction   
D3FEND complements ATT&CK by focusing on defensive techniques.

---

## Trap

“Enumeration is passive.”

Why Confusing   
Students associate all information gathering with passive recon.

Correct Distinction   
Enumeration actively queries services and is more intrusive.

# 10. Micro Facts (Exam Boosters)

* Reconnaissance is usually the first hacking phase.
* Enumeration is more intrusive than scanning.
* ATT&CK = adversary behavior matrix.
* D3FEND = defensive countermeasure mapping.
* Cyber Kill Chain has seven phases.
* PTES begins with pre-engagement interactions.
* OWASP focuses on web application security testing.
* OSSTMM emphasizes measurable testing methodology.
* NIST CSF contains five core functions.
* Maintaining access commonly uses persistence mechanisms.
* Clearing tracks involves anti-forensics techniques.
* Active reconnaissance is easier to detect.
* Passive reconnaissance heavily relies on OSINT.
* C2 traffic often appears as outbound beaconing.
* Enumeration frequently targets SMB, SNMP, LDAP, and DNS.
* Banner grabbing helps identify software versions.
* ATT&CK maps real-world TTPs.
* Diamond Model includes adversary, infrastructure, capability, and victim.
* AI-assisted ethical hacking improves automation and threat analysis.
* Reporting is a required ethical hacking phase.
