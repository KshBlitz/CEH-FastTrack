# 2. Core Concepts

## Information Security Controls

Concept → Safeguards or countermeasures implemented to reduce security risks and vulnerabilities        
Purpose → Protect confidentiality, integrity, and availability (CIA)        
Key Detail → Controls may be administrative, technical, or physical

Concept → Defense-in-Depth      
Purpose → Layered security architecture     
Key Detail → Uses multiple overlapping controls so failure of one control does not compromise the system

Concept → Risk Management       
Purpose → Identify, assess, prioritize, and mitigate risks      
Key Detail → Risk cannot be eliminated completely; it can only be reduced, transferred, avoided, or accepted

Concept → Incident Management       
Purpose → Detect, respond to, contain, and recover from security incidents      
Key Detail → Includes preparation, detection, containment, eradication, recovery, and lessons learned

Concept → Identity and Access Management (IAM)      
Purpose → Ensure authorized users gain appropriate access to resources      
Key Detail → Includes authentication, authorization, accounting, provisioning, SSO, MFA, federation, and privileged access management

Concept → Business Continuity Planning (BCP)        
Purpose → Maintain critical business operations during disruptions      
Key Detail → Focuses on continuity of operations

Concept → Disaster Recovery Planning (DRP)      
Purpose → Restore IT infrastructure and services after disaster     
Key Detail → Focuses on recovery of systems, applications, and data

Concept → Data Loss Prevention (DLP)        
Purpose → Prevent leakage, exfiltration, or unauthorized disclosure of sensitive data       
Key Detail → Monitors data at rest, in motion, and in use

Concept → Security Awareness Training       
Purpose → Reduce human-related security risks       
Key Detail → Addresses phishing, social engineering, password security, insider threats, and safe usage practices

Concept → AI/ML in Security     
Purpose → Improve threat detection, automation, anomaly detection, and predictive analysis      
Key Detail → Used in IDS/IPS, UEBA, malware analysis, fraud detection, and SOC automation

# 3. Definitions & Classifications

## Types of Security Controls

Category → Administrative Controls      
Characteristics → Policies, procedures, governance, training, risk assessment, background checks        
Example → Security policy, acceptable use policy (AUP), NDA, security awareness training

Category → Technical (Logical) Controls     
Characteristics → Implemented through hardware/software mechanisms      
Example → Firewalls, IDS/IPS, ACLs, MFA, antivirus, encryption

Category → Physical Controls        
Characteristics → Protect physical assets and facilities        
Example → CCTV, biometric locks, fences, mantraps, security guards

---

## Functional Classification of Controls

Type → Preventive Controls
Characteristics → Prevent security incidents before occurrence
Example → Firewall, MFA, IPS, least privilege, access control

Type → Detective Controls
Characteristics → Detect ongoing or completed attacks
Example → IDS, SIEM, audit logs, CCTV, file integrity monitoring

Type → Corrective Controls
Characteristics → Fix or mitigate damage after incident
Example → Patch management, malware removal, configuration correction

Type → Recovery Controls
Characteristics → Restore operations after incident
Example → Backups, disaster recovery systems, redundancy

Type → Deterrent Controls
Characteristics → Discourage attackers from attempting attacks
Example → Warning banners, visible CCTV cameras, legal notices

Type → Compensating Controls
Characteristics → Alternative controls when primary controls are not feasible
Example → Increased monitoring when MFA cannot be implemented

Type → Directive Controls
Characteristics → Direct user behavior
Example → Security policies, standards, procedures

---

## Access Control Models

Model → Discretionary Access Control (DAC)
Characteristics → Owner determines permissions
Example → Windows NTFS permissions

Model → Mandatory Access Control (MAC)
Characteristics → Central authority controls access using labels/classifications
Example → Military/government classified systems

Model → Role-Based Access Control (RBAC)
Characteristics → Access assigned according to job role
Example → HR role, finance role, administrator role

Model → Rule-Based Access Control
Characteristics → Access based on predefined rules/conditions
Example → Firewall ACLs

Model → Attribute-Based Access Control (ABAC)
Characteristics → Uses attributes such as department, location, time, device state
Example → Context-aware access systems

---

## Authentication Factors

Factor → Something You Know
Characteristics → Knowledge-based authentication
Example → Password, PIN

Factor → Something You Have
Characteristics → Possession-based authentication
Example → Smart card, OTP token

Factor → Something You Are
Characteristics → Biometric authentication
Example → Fingerprint, iris scan, facial recognition

Factor → Somewhere You Are
Characteristics → Location-based authentication
Example → GPS/IP geolocation

Factor → Something You Do
Characteristics → Behavioral biometrics
Example → Typing rhythm, signature dynamics

---

## Risk Management Terms

Term → Asset
Meaning → Anything valuable to organization
Example → Data, servers, employees, intellectual property

Term → Threat
Meaning → Potential cause of harm
Example → Malware, insider threat, ransomware

Term → Vulnerability
Meaning → Weakness exploitable by attacker
Example → Unpatched system, weak password

Term → Risk
Meaning → Probability of threat exploiting vulnerability
Formula → Risk = Threat × Vulnerability × Impact

Term → Exposure
Meaning → State of being vulnerable to damage/loss

Term → Countermeasure
Meaning → Safeguard implemented to reduce risk

---

## Risk Treatment Strategies

Strategy → Risk Avoidance
Characteristics → Eliminate risky activity
Example → Disabling vulnerable service

Strategy → Risk Reduction/Mitigation
Characteristics → Reduce likelihood or impact
Example → Implement firewall and IDS

Strategy → Risk Transfer
Characteristics → Shift risk to third party
Example → Cyber insurance, outsourcing

Strategy → Risk Acceptance
Characteristics → Accept residual risk
Example → Low-impact vulnerability accepted

# 4. Differentiation Blocks

## BCP vs DRP

BCP → Maintains business operations during disruption
DRP → Restores IT infrastructure after disruption

BCP → Business-focused
DRP → Technology-focused

BCP → Ensures continuity
DRP → Ensures recovery

---

## IDS vs IPS

IDS → Detects attacks
IPS → Detects and blocks attacks

IDS → Passive monitoring
IPS → Inline active protection

IDS → Generates alerts
IPS → Drops/block packets

---

## Authentication vs Authorization vs Accounting

Authentication → Verifies identity
Authorization → Determines permissions
Accounting → Tracks activities/logging

Mnemonic → AAA

---

## Vulnerability vs Threat vs Risk

Vulnerability → Weakness
Threat → Potential danger
Risk → Probability of exploitation causing impact

---

## Least Privilege vs Need to Know

Least Privilege → Minimum permissions necessary
Need to Know → Access only to information required for job

---

## Hardening vs Patch Management

Hardening → Reduce attack surface by disabling unnecessary services/features
Patch Management → Apply vendor fixes/updates for vulnerabilities

---

## Symmetric vs Asymmetric Encryption

Symmetric → Same key for encryption/decryption
Asymmetric → Public/private key pair

Symmetric → Faster
Asymmetric → More secure key distribution

# 8. Scenario / Attack Recognition

Scenario/Behavior → Likely Concept or Attack

Organization implements multiple overlapping security controls at network, endpoint, and application layers → Defense-in-Depth

User authenticates with password + fingerprint → Multi-Factor Authentication (MFA)

Security team receives alerts but traffic is not blocked → IDS deployment

Traffic is automatically dropped when malicious patterns detected → IPS deployment

Attackers move laterally after compromising single system because network lacks segmentation → Weak defense-in-depth/network segmentation

Company restores operations from backups after ransomware attack → Disaster Recovery Process

Employee retains permissions from previous departments → Privilege Creep

Users access all applications using one login session → Single Sign-On (SSO)

Security system identifies abnormal login behavior from impossible geographic locations → UEBA/AI-driven anomaly detection

System blocks USB copying of confidential files → DLP enforcement

Attacker compromises account due to weak password reuse → Weak authentication controls

Unauthorized employee accesses payroll records because permissions assigned incorrectly → Authorization failure

Security operations center correlates logs from multiple devices to detect attacks → SIEM usage

# 9. MCQ Traps & Confusions

Trap → IDS and IPS treated as same
Why Confusing → Both monitor traffic
Correct Distinction → IDS detects/alerts; IPS blocks/prevents

---

Trap → Authentication confused with authorization
Why Confusing → Both involve access
Correct Distinction → Authentication verifies identity; authorization determines permissions

---

Trap → BCP confused with DRP
Why Confusing → Both related to disaster scenarios
Correct Distinction → BCP maintains operations; DRP restores infrastructure

---

Trap → Risk elimination assumption
Why Confusing → Security controls reduce risk significantly
Correct Distinction → Residual risk always remains

---

Trap → MAC confused with DAC
Why Confusing → Both are access control models
Correct Distinction → MAC controlled centrally using labels; DAC controlled by owner

---

Trap → Preventive vs detective controls
Why Confusing → Same tool may provide both functions
Correct Distinction → Preventive stops attacks; detective identifies attacks/events

---

Trap → Encryption vs hashing
Why Confusing → Both transform data
Correct Distinction → Encryption reversible using key; hashing one-way integrity function

---

Trap → Hot site vs warm site vs cold site
Why Confusing → All are recovery sites
Correct Distinction → Hot = fully operational; Warm = partially configured; Cold = infrastructure only

---

Trap → Vulnerability vs exploit
Why Confusing → Closely related terms
Correct Distinction → Vulnerability is weakness; exploit is method/code used to abuse weakness

---

Trap → Least privilege vs separation of duties
Why Confusing → Both reduce insider abuse
Correct Distinction → Least privilege minimizes permissions; separation of duties divides responsibilities

* Incident response includes lessons learned phase.
* UEBA uses AI/ML to identify anomalous behavior.
* Patch management reduces exploitability window.

* Hashing primarily provides integrity.
