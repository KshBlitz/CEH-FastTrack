# Information Security Overview — CEH Exam Study Sheet

# 1. Information Security Concepts

Information security refers to the **protection and safeguarding of information and information systems** from:

* unauthorized access
* disclosure
* alteration
* destruction
* disruption

If sensitive information falls into the wrong hands, organizations may suffer:

* financial loss
* brand/reputation damage
* legal consequences
* operational disruption

Information security focuses on protecting **critical information resources**. 

---

# 2. Elements of Information Security

CEH defines information security as the state in which:

> “the possibility of theft, tampering, disruption of information and services is kept low or tolerable.”

The five major elements are:

| Element         | Purpose                            |
| --------------- | ---------------------------------- |
| Confidentiality | Prevent unauthorized disclosure    |
| Integrity       | Prevent unauthorized modification  |
| Availability    | Ensure systems/data are accessible |
| Authenticity    | Verify genuineness of users/data   |
| Non-Repudiation | Prevent denial of actions/messages |



---

## Confidentiality

Confidentiality ensures information is accessible **only to authorized users**.

### Confidentiality Breaches May Occur Due To:

* improper data handling
* hacking attempts
* weak controls

### Confidentiality Controls

* data classification
* encryption
* proper disposal of storage devices

  * DVDs
  * USB drives
  * Blu-ray discs

Key exam point:

* Confidentiality mainly protects against **unauthorized disclosure**.



---

## Integrity

Integrity ensures data is:

* accurate
* trustworthy
* unaltered

Protects against:

* improper modification
* unauthorized changes

### Integrity Mechanisms

* checksum
* hashing
* access control

Access control ensures only authorized users can:

* update
* add
* delete data

Key exam distinction:

* Integrity = protection from modification
* Confidentiality = protection from disclosure



---

## Availability

Availability ensures systems and data are:

* accessible
* usable
* operational when needed

### Availability Protection Measures

* redundant systems
* clustered machines
* antivirus software
* DDoS prevention systems

Key exam point:

* Availability attacks commonly include:

  * DoS
  * DDoS



---

## Authenticity

Authenticity verifies that:

* communications are genuine
* data is not corrupted
* entities are legitimate

### Common Authenticity Controls

* biometrics
* smart cards
* digital certificates
* authentication systems



---

## Non-Repudiation

Non-repudiation guarantees:

* sender cannot deny sending a message
* receiver cannot deny receiving it

### Common Mechanism

* digital signatures

Key exam association:

* Non-repudiation ↔ Digital signatures



---

# 3. Information Security Attacks: Motives, Goals, and Objectives

An attack is:

> “an action performed with the intent to breach an IT system’s security.”

Attack formula emphasized in CEH:

```text
Attack = Motive (Goal) + Method (TTP) + Vulnerability
```



---

## Common Motives Behind Attacks

### Financial Motives

* information theft
* ransomware
* fraud

### Political/Ideological Motives

* propaganda
* hacktivism
* cyber warfare

### Operational Disruption

* disrupting business continuity
* damaging critical infrastructure

### Personal Motives

* revenge
* fame
* curiosity
* peer recognition

### State-Level Motives

* military objectives
* espionage

Important CEH examples:

* bring financial loss to target
* damage reputation
* demand ransom
* propagate religious/political beliefs

# 4. Tactics, Techniques, and Procedures (TTPs)

TTPs describe patterns of adversary behavior.

| Component  | Meaning                           |
| ---------- | --------------------------------- |
| Tactics    | Overall strategy/objective        |
| Techniques | Methods used to achieve objective |
| Procedures | Exact implementation steps        |



---

## Important Distinctions

### Tactics

Describe:

* “what” attacker wants to achieve

Examples:

* credential theft
* persistence
* privilege escalation

---

### Techniques

Describe:

* “how” attacker achieves objective

Examples:

* phishing
* SQL injection
* password spraying

---

### Procedures

Describe:

* exact execution details

Examples:

* specific commands
* scripts
* malware deployment sequence

---

## Why TTPs Matter

TTP analysis helps defenders:

* understand attacker behavior
* identify vulnerabilities
* implement defensive measures
* predict future attacks

CEH exam association:

* TTPs are heavily linked with:

  * threat intelligence
  * MITRE ATT&CK
  * incident analysis



---

# 5. Vulnerability

A vulnerability is:

> “a weakness in the design or implementation of a system that can be exploited.”

It enables attackers to:

* bypass security
* gain unauthorized access
* compromise systems



---

# 6. Common Reasons for Vulnerabilities

## 1. Hardware or Software Misconfiguration

Examples:

* insecure protocols
* incorrect settings
* exposed services

Example from CEH:

* using unencrypted protocols

---

## 2. Insecure or Poor Design

Examples:

* insecure VPN deployment
* weak architecture
* insecure application logic

---

## 3. Inherent Technology Weaknesses

Examples:

* outdated operating systems
* unpatched software
* insecure protocols

---

## 4. End-User Carelessness

Examples:

* weak passwords
* clicking malicious links
* connecting to insecure networks

---

## 5. Intentional End-User Acts

Example:

* employee intentionally exposing company information



---

# 7. Examples of Vulnerabilities

## Technological Vulnerabilities

### TCP/IP Protocol Vulnerabilities

Protocols identified as inherently insecure:

* HTTP
* FTP
* ICMP
* SNMP
* SMTP

### Operating System Vulnerabilities

* inherent insecurity
* lack of patches
* unpatched latest updates

### Network Device Vulnerabilities

* weak passwords
* lack of authentication
* insecure routing protocols
* firewall vulnerabilities



---

## Configuration Vulnerabilities

| Vulnerability                     | Description                                                |
| --------------------------------- | ---------------------------------------------------------- |
| User account vulnerabilities      | Weak/insecure transmission of usernames/passwords          |
| System account vulnerabilities    | Weak passwords for system accounts                         |
| Internet service misconfiguration | Improper IIS, Apache, FTP, Terminal Services configuration |
| Default passwords/settings        | Leaving vendor defaults unchanged                          |
| Network device misconfiguration   | Incorrect router/switch/firewall settings                  |

# 8. Classification of Attacks

According to IATF, attacks are classified into five categories:

1. Passive attacks
2. Active attacks
3. Close-in attacks
4. Insider attacks
5. Distribution attacks



---

# 9. Passive Attacks

Passive attacks involve:

* monitoring
* intercepting
* eavesdropping

Goal:

* gather information without altering systems

Characteristics:

* difficult to detect
* no direct interaction with target

### Examples

* footprinting
* sniffing/eavesdropping
* traffic analysis
* decrypting weakly encrypted traffic

Key exam distinction:

* Passive attacks do NOT modify data.



---

# 10. Active Attacks

Active attacks:

* modify data
* disrupt communication
* exploit vulnerabilities

Characteristics:

* detectable
* involve direct interaction

### Examples

* DoS attacks
* malware attacks
* spoofing
* replay attacks
* password attacks
* session hijacking
* MITM attacks
* DNS poisoning
* compromised-key attacks
* firewall/IDS attacks
* privilege escalation
* SQL injection
* XSS
* directory traversal
* exploitation of application/OS flaws

Key exam distinction:

* Active attacks alter systems or services.



---

# 11. Close-In Attacks

Close-in attacks require physical proximity.

Goals:

* gather information
* modify information
* gain access

### Methods

* shoulder surfing
* eavesdropping
* dumpster diving
* social engineering



---

# 12. Insider Attacks

Performed by trusted individuals with legitimate access.

Targets:

* confidential information
* business operations
* reputation
* profit

### Examples

* wiretapping
* theft of devices
* social engineering
* data theft
* pod slurping
* planting keyloggers/backdoors/malware

Key exam point:

* Insider threats are difficult to detect.



---

# 13. Distribution Attacks

Occur when attackers tamper with:

* hardware
* software
* supply chain components

Timing:

* during manufacturing
* during distribution
* during transit

### Examples

* modifying software before delivery
* modifying hardware during production

Modern association:

* supply chain attacks



---

# 14. Information Warfare

Information Warfare (InfoWar) refers to:

> “the use of information and communication technologies (ICT) to gain competitive advantage over opponents.”



---

# 15. Types of Information Warfare

## Command and Control Warfare (C2 Warfare)

Goal:

* deny information to opponent
* protect own information systems

---

## Intelligence-Based Warfare

Uses:

* sensor-based technologies
* information dominance

---

## Electronic Warfare

Uses:

* electromagnetic spectrum
* cryptographic/electronic techniques

---

## Psychological Warfare

Uses:

* propaganda
* terror
* demoralization

---

## Hacker Warfare

Uses:

* viruses
* worms
* Trojan horses
* logic bombs
* trap doors
* sniffers

---

## Economic Warfare

Targets:

* financial systems
* economy
* business operations

---

## Cyberwarfare

Defined as:

* attacks against information systems
* attacks on virtual entities/groups

Examples:

* semantic attacks
* network attacks
* military cyber operations



---

# 16. Defensive vs Offensive Information Warfare

| Defensive Information Warfare | Offensive Information Warfare |
| ----------------------------- | ----------------------------- |
| Protect ICT assets            | Attack ICT assets             |
| Prevention                    | Web application attacks       |
| Detection                     | Malware attacks               |
| Emergency response            | MITM attacks                  |
| Recovery                      | System hacking                |
